# 変更要求票 — issue #2

## 入口種別

issue直（(a) GitHub issue。PrimeBrains/moira#2「普遍訂正原則を正典化する — moira-model-update サイクル」）

## 明確化した変更要求文

issue PrimeBrains/sdd-workshop#33 の分析（一次資料: `moira/analysis/issue-33-irreversibility.md`）で
2026-07-18 にユーザー合意済みの**普遍訂正原則**——

> すべてのイベントは訂正可能。ただし (a) 追記のみ（訂正それ自体が履歴に残る）(b) reason 必須
> (c) 元イベント id 名指し (d) 訂正は計器に見える（隠れて直せない）(e) 対象は記録であって事実ではない

——を、`moira-model-update` ゲート（独立敵対者による多重レビューループ・falsifiable 確定ゲート）を通して
**`moira/MODEL.md` に正典化**する。具体的な改定議題は次の 9 点（一次資料 §4-§5）:

1. **§7#19（既知の残余）の置き換え** — cost 誤計上・誤 cancel の「訂正イベント無し」裁定を普遍訂正原則で更新する。
2. **A6 射程の明確化** — A6 非負が守るのは事実の存在論（負の投下時間は無い）であり、記録の訂正はその外、を明文化する。負値 cost（マイナス相殺イベント）は導入しない。
3. **I4 完了施錠の格下げ** — 「不可」から「訂正すると大きく音が鳴る」（reason 必須＋計器常設表示。統制された再ベースラインと同型）へ改める。
4. **cancelled 訂正の意味論** — un-cancel 専用機構は作らず「cancel は誤記だった」という記録訂正として表現する。導出層の再計算規則を定める。裸の逆行遷移（訂正の体裁を持たない書き込み）は引き続き書き込み層で塞ぐ側。
5. **訂正の表現形の裁定** — 第二層ファイル（R-U14 同型。一次資料の現時点推奨）vs 第 5 イベント型。§5「イベント 4 型」方針との折り合いを裁定する。
6. **ゲーミング耐性の計器設計** — 訂正件数・施錠対象への訂正を health のどこへどう常設表示するか（閾値・色分けを裁量ノブにしない工夫を含む）。
7. **歴史の読み直しの裁定** — 訂正は現在の射影のみ・保存済みレポートは当時の文書として不変、で足りるか。bitemporal（起きた時刻×知った時刻）の要否。#37（sdd-workshop）で暫定導入した backfill 拒否の恒久裁定を含む。
8. **訂正の連鎖規則** — 訂正記録自体が誤りだった場合の規則（latest-wins か明示 supersede か）。
9. **付随整備の裁定** — cost/relate への reason 欄・assignee/reviewer 解除センチネル（§7#16(c)/#18(d)）・所属の矛盾主張への警告（R-U12 同型）。
