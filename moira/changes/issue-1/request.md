## 明確化した変更要求文

DECISIONS-CATALOG.md の各設計判断エントリから**裏面（根拠 clause ID・関連実装 ref の対応表部分）を削除**し、
**表面（場面・決めたこと・代替・賭け金・判定文・仕分けの平易列）のみを正典として維持**する。あわせて:

1. **計器⑥の照合対象特定の置換**: `decision-conformance` スキルおよび `decision-conformance-checker`
   エージェント定義における照合対象の特定方法を、「裏面 ref 起点（目録の ref 欄・逆引き grep）」から
   「**AI の意味突合**（判断文そのものを起点に、実装・spec への意味検索で対応を毎回導出する）」へ書き換える。
2. **被覆マップ（計器割付）の扱いの設計**: 裏面と同居している被覆情報（どの判断がどの計器で照合されるか）に
   ついて、(a) 最小の表として存続させる か (b) 派生ビュー化（毎回導出）する かを決めて反映する。
3. **参照記述の同期**: `.kiro/steering/moira-verification.md` の裏面・被覆マップ参照記述、および
   `.kiro/steering/moira-change-management.md` §2（P2 トレース機構の「裏面 ref と計器被覆マップ」「変更 path →
   裏面 ref の逆引き」）・§5（逆引きの実行定義・正直枠・「その issue の確定後に本記述を更新する」の予告部分）を、
   裏面削除後の姿に更新する。

背景の裁定（2026-07-19・人間）: 当初案の「機械可読 path 欄を足す正規化」ではなく**裏面自体の削除**へ方向転換
——手で維持する対応表（ドリフト面）を増やさず、対応の導出を毎回 AI に任せる。正典は判断文そのもの。
トレードオフ（照合ごとの意味検索コスト増 ↔ 維持コストゼロ）は #40 の経済学と同じ賭けとして受け入れ済み。
